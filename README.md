# Betelgeuse — Інтелектуальна система автономної посадки ракетоносія

## 🎯 Мета проєкту
Розробка та порівняльне дослідження інтелектуальних систем керування посадкою ракети-носія в умовах невизначеності.  
Проєкт реалізовано в рамках **магістерської дипломної роботи** за темою:

> **«Розроблення інтелектуальної системи автономної посадки ракетоносія на основі нечіткої логіки та машинного навчання»**

## 📋 Опис
Симуляційна платформа на Unity, що дозволяє:
- Моделювати точну фізику польоту та посадки ракети (метод Рунге-Кутта 4-го порядку)
- Порівнювати три алгоритми керування: **PID**, **Fuzzy Logic (Sugeno)** та **Neural Network** з еволюційним навчанням
- Проводити серії Monte-Carlo експериментів з урахуванням випадкових збурень (бічний вітер, варіація маси палива, початковий кут нахилу)
- Оцінювати якість посадки за комплексною метрикою (швидкість торкання, кут нахилу, залишок палива)

## 🧠 Режими керування

| Режим              | Опис                                                                 | Переваги |
|--------------------|----------------------------------------------------------------------|----------|
| **PID**            | Класичний PID-регулятор з окремим каналом керування вертикальною швидкістю | Простота, висока стабільність |
| **Fuzzy Logic**    | Нечітка логіка Сугено з 7 нечіткими правилами                        | Інтерпретованість, хороша реакція на критичні ситуації |
| **Neural Network** | Проста нейромережа (4 входи → 1 прихований → 1 вихід) з онлайн-навчанням методом мутацій | Адаптивність, здатність покращуватись з часом |

## ⚙️ Ключові особливості реалізації

- **Точна фізика**: інтеграція руху методом **Рунге-Кутта 4-го порядку (RK4)**
- **Реалістична модель атмосфери** та залежна від висоти гравітація
- **Моделювання кутового руху** з урахуванням моменту сил від відхиленого вектора тяги
- **Monte-Carlo тестування** — накладання випадкового шуму для оцінки стійкості алгоритмів
- **Онлайн-навчання** нейромережі після кожної посадки (збереження найкращих ваг)
- **Повне логування**: траєкторії у CSV + автоматична порівняльна таблиця результатів
- **Візуалізація**: LineRenderer траєкторії, системи частинок полум’я та диму, HUD телеметрії в реальному часі

## 📁 Структура файлів (відповідно до проєкту)

```
Assets/Scripts/
├── Control/
│   ├── FuzzyLandingController.cs     # Нечіткий контролер посадки (Sugeno)
│   ├── NeuralController.cs           # Нейромережевий контролер + еволюційне навчання
│   └── PIDController.cs              # Універсальний PID-регулятор з анти-інтегральним віндапом
├── Core/
│   ├── AtmosphereModel.cs            # Статична модель атмосфери та гравітації
│   ├── DataLogger.cs                 # Логування даних симуляції у CSV
│   ├── LandingMetrics.cs             # Метрики успішності посадки та розрахунок загальної оцінки
│   ├── RocketPhysics.cs              # Головний компонент (фізика + вибір режиму керування)
│   ├── RocketState.cs                # Структура повного фізичного стану ракети
│   └── SimulationManager.cs          # Менеджер серійних Monte-Carlo експериментів
├── Parameters/
│   └── SimulationParameters.cs       # ScriptableObject з початковими умовами та характеристиками
├── UI/
│   ├── ExperimentDashboard.cs        # UI-панель для керування експериментами
│   ├── TelemetryHUD.cs               # Відображення телеметрії (висота, швидкість, тяга, кут)
│   └── TrajectoryVisualizer.cs       # Візуалізація траєкторії польоту
└── Utils/
    └── CameraFollow.cs               # Плавне слідкування камери за ракетою
```

## 🚀 Як запустити проєкт

1. Відкрийте проєкт у **Unity 2022.3 або новіше**
2. Скопіюйте всю папку `Scripts` (разом з підпапками `Control`, `Core`, `Parameters`, `UI`, `Utils`) у теку `Assets/`
3. Створіть файл параметрів:
   - `Create → Betelgeuse → Simulation Parameters`
4. Створіть префаб ракети та додайте на нього необхідні компоненти з відповідних папок:
   - `Core/RocketPhysics`
   - `Control/FuzzyLandingController`
   - `Control/NeuralController`
   - `Core/DataLogger`
   - `UI/TrajectoryVisualizer` (обов’язково з компонентом `LineRenderer`)
   - `Utils/CameraFollow`
   - `UI/TelemetryHUD` (з посиланнями на `TMP_Text`)
5. У компоненті `RocketPhysics` призначте створений `SimulationParameters`
6. Для запуску повного порівняльного експерименту:
   - Поставте галочку `runFullExperiment` у `SimulationManager`
   - Натисніть **Play**

## 📊 Критерії успішної посадки

Посадка вважається успішною за одночасного виконання умов:
- Вертикальна швидкість торкання < **3.5 м/с**
- Кут відхилення від вертикалі при посадці < **7°**

**Загальна оцінка (Success Score)** розраховується за формулою:
```
Score = (0.5 × velScore + 0.3 × angleScore + 0.2 × fuelScore) × 100
```

## 📈 Очікувані результати

При проведенні серій з 25+ тестів з увімкненим шумом зазвичай спостерігається:
- **Fuzzy Logic** — найвища успішність у складних умовах
- **Neural Network** — поступове покращення показників завдяки онлайн-навчанню
- **PID** — стабільна, але менш гнучка поведінка при сильних збуреннях
